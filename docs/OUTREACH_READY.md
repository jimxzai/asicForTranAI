## üéì Email Best Practices

### Do's ‚úÖ
- **Keep it concise**: 300-500 words max
- **Lead with value**: What's in it for them?
- **Include attachment**: PDF overview (don't make them search)
- **Clear ask**: Developer access OR research collaboration
- **Timeline**: NeurIPS 2026 creates urgency
- **Follow up**: If no response in 5 business days

### Don'ts ‚ùå
- **Don't oversell**: Avoid "revolutionary" or "game-changing"
- **Don't rush**: Give them 3-5 days to respond
- **Don't multi-track**: Don't CC multiple departments
- **Don't attach code**: Keep it professional (link to GitHub only)
- **Don't demand**: Request, don't require

### Subject Line Testing
Original: "Formally Verified 3.5-bit LLM Inference for Groq LPU"

Alternatives (if no response):
- "NeurIPS 2026: Verified LLM Inference on Groq LPU"
- "Safety-Critical LLM Partnership - Groq Developer Access"
- "177 GB LLaMA-405B on Cerebras WSE (All On-Chip)"
